## ADDED Requirements

### Requirement: Serif heading typefaces
The site SHALL use elegant serif typefaces for all headings.

#### Scenario: Headings establish character
- **WHEN** any heading displays
- **THEN** the typeface is a serif font that conveys warmth, nostalgia, and literary quality

### Requirement: Readable body text
The site SHALL use highly readable typefaces for body text optimized for long-form reading.

#### Scenario: Body text is comfortable to read
- **WHEN** body text displays
- **THEN** the typeface prioritizes readability over decoration with appropriate x-height and character spacing

### Requirement: Long-form content optimization
The site SHALL optimize typography for long-form content with appropriate line length, line height, and paragraph spacing.

#### Scenario: Long articles are comfortable
- **WHEN** a visitor reads a full episode review
- **THEN** line length is 60-75 characters, line height is 1.6-1.8, and paragraph spacing provides visual breaks

### Requirement: Type scale hierarchy
The site SHALL establish a clear type scale for heading levels and body text.

#### Scenario: Type scale is consistent
- **WHEN** different heading levels and text elements display
- **THEN** they follow a consistent size hierarchy (e.g., H1 > H2 > H3 > body > small)

### Requirement: Responsive typography
The site SHALL adjust typography sizes appropriately across device sizes.

#### Scenario: Typography scales responsively
- **WHEN** the site displays on different screen sizes
- **THEN** font sizes scale appropriately ensuring readability on mobile without being too large on desktop

### Requirement: Emphasis and formatting support
The site SHALL support emphasis (italic), strong (bold), and other formatting in body text.

#### Scenario: Text formatting renders correctly
- **WHEN** MDX content includes bold, italic, or other formatting
- **THEN** the formatting displays with appropriate styles that maintain readability
